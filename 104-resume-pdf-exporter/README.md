# 104 Resume PDF Exporter 使用說明

> 套件名稱：**104 VIP Resume PDF Exporter (Control)**  
> 版本：**v1.4.2**  
> 用途：在 104 VIP 網站中，將履歷頁自動轉成 **PDF** 並下載，可支援 **單頁下載** 與 **從目前分頁往右批次下載**。

---

## 1. 這個擴充功能可以做什麼？

這個套件是專門給 104 招募作業使用的履歷 PDF 匯出工具。它可以：

- **下載目前履歷**：直接把目前的 104 履歷轉成 PDF 並下載
- **往右批次下載**：從目前分頁開始，往右依序下載多份履歷 PDF
- **不需要先手動開列印頁**：套件會自動在背景開啟預覽頁並匯出 PDF
- **支援多種履歷來源頁**：搜尋頁、文件頁、預覽頁、主動投遞 / 應徵頁
- **可設定檔名前綴 / 後綴**：方便加上職稱、來源等資訊
- **可設定下載子資料夾**：便於集中管理履歷 PDF
- **顯示進度與狀態**：包含執行中進度、成功與失敗狀態

---

## 2. 支援頁面類型

此套件支援以下 104 VIP 相關頁面：

- `https://vip.104.com.tw/ResumeTools/resumePreview?...`
- `https://vip.104.com.tw/search/*`
- `https://vip.104.com.tw/document/*`
- 某些主動投遞 / 應徵情境頁面（例如 `ApplyResume`）

也就是說，你不一定要先切到預覽頁；在可辨識的履歷頁上即可直接操作。

## 3. 安裝方式（Chrome 載入未封裝擴充功能）

### 步驟 1：先解壓縮檔案

請先把壓縮檔解壓縮，例如：

- `104-resume-pdf-exporter.zip`

解壓後，請找到真正的擴充功能資料夾，裡面至少要有：

- `manifest.json`
- `background.js`
- `popup.html`
- `popup.js`

### 步驟 2：打開 Chrome 擴充功能頁

在 Chrome 網址列輸入：

```text
chrome://extensions/
```

### 步驟 3：開啟開發人員模式

將右上角的 **「開發人員模式」** 開啟。

### 步驟 4：載入未封裝項目

按下：

- **「載入未封裝項目」**

然後選擇擴充功能資料夾。  
請注意，如果你的解壓縮結果有多包一層資料夾，要選到 **含有 `manifest.json` 的那一層**。

### 步驟 5：確認安裝完成

安裝成功後，Chrome 會顯示：

- **104 VIP Resume PDF Exporter (Control)**

建議將它釘選到工具列，以便快速操作。

## 4. 安裝後第一次必做設定

此套件會使用 Chrome 下載功能把 PDF 存到你的電腦，因此建議先調整 Chrome 的下載設定。

## 4.1 建議關閉「每次下載前詢問儲存位置」

請到 Chrome 設定中的下載區：

1. 打開 Chrome 設定
2. 搜尋「下載」
3. 找到下載相關設定
4. **關閉**：`每次下載前詢問儲存位置`

### 為什麼要關閉？

因為如果每下載一份履歷都跳出另存視窗，批次下載就會被中斷，無法自動完成。

---

## 4.2 設定 Chrome 預設下載資料夾

你可以在 Chrome 設定中設定一個預設下載資料夾，例如：

```text
/Users/你的帳號/Downloads
```

再搭配此擴充功能的「下載子資料夾」設定，例如：

```text
招募/104履歷下載區/
```

這樣最終 PDF 會進到：

```text
Chrome預設下載資料夾/招募/104履歷下載區/
```

---

## 5. Popup 介面欄位說明

打開擴充功能後，會看到以下設定欄位：

### 5.1 下載目前履歷

將目前這個履歷頁匯出為 PDF。

### 5.2 往右批次下載

從目前分頁開始，往右依序處理同一個 Chrome 視窗中的履歷分頁。

### 5.3 下載子資料夾

表示下載到 **Chrome 預設下載資料夾底下的哪個子資料夾**。  
例如輸入：

```text
104履歷下載區/
```

或：

```text
招募/104履歷下載區/
```

不填則直接下載到 Chrome 預設下載資料夾。

### 5.4 檔名前綴

可填入職稱或分類，例如：

```text
Java工程師
```

### 5.5 檔名後綴

可填入來源或補充資訊，例如：

```text
104
```

### 5.6 首份等待（ms）

第一次開背景預覽頁後，額外等待的時間。  
預設較長，避免第一份尚未完整渲染就開始匯出。

### 5.7 後續等待（ms）

第二份之後每一份的等待時間。  
如果你的網路或 104 頁面比較慢，可適度調高。

### 5.8 儲存設定

將目前 Popup 中的設定保存到本機，下次打開時會延續使用。

---

## 6. 檔名規則說明

此套件輸出的 PDF 檔名格式如下：

```text
職稱_Name_來源.pdf
```

例如：

```text
Java工程師_王小明_104.pdf
```

若前綴與後綴都留白，則會變成：

```text
王小明.pdf
```

若只填前綴：

```text
Java工程師_王小明.pdf
```

若只填後綴：

```text
王小明_104.pdf
```

## 7. 使用教學

## 7.1 下載目前履歷

適合情境：只想把當前這一位候選人的履歷下載成 PDF。

### 操作步驟

1. 打開一個 104 履歷頁
2. 點擊擴充功能圖示
3. 確認設定（下載子資料夾、檔名前綴、檔名後綴）
4. 點 **「下載目前履歷」**
5. 套件會自動：
   - 判斷目前頁面資訊
   - 在背景開啟履歷預覽頁
   - 使用 PDF 匯出功能產生 PDF
   - 下載到指定資料夾
   - 關閉背景預覽頁

### 適合使用情境

- 臨時需要某位候選人 PDF 履歷
- 補下載單一人選
- 需要快速附檔給主管或保存本地檔案

---

## 7.2 往右批次下載

適合情境：你一次開了很多份履歷頁，想批次下載成 PDF。

### 操作步驟

1. 在同一個 Chrome 視窗中，將多份履歷頁由左到右排好
2. 把目前分頁停在第一份要下載的履歷
3. 點開擴充功能
4. 檢查設定是否正確
5. 點選 **「往右批次下載」**
6. 套件會從目前分頁開始，往右逐一處理
7. 遇到第一個非履歷頁時停止
8. 每份履歷會自動輸出成 PDF 並下載

### 批次規則

- 只會處理 **目前這頁與右側的分頁**
- 會在遇到 **第一個非 104 履歷頁** 時停止
- 支援大小寫不同的履歷路徑，例如：
  - `/search/searchResumeMaster`
  - `/search/SearchResumeMaster`
- 也支援部分主動投遞 / 應徵頁，會嘗試解析出真正的履歷網址

---

## 8. 建議操作流程

### 情境 A：單一下載

1. 開啟候選人履歷頁
2. 輸入檔名前綴，例如職缺名稱
3. 按 **下載目前履歷**

### 情境 B：一次下載一整批同職缺履歷

1. 把同一職缺的履歷頁都開好
2. 依順序排列在同一視窗
3. 前綴填入職缺名稱，例如 `SA系統分析師`
4. 後綴填入來源，例如 `104`
5. 按 **往右批次下載**

這樣得到的檔案會像：

```text
SA系統分析師_王小明_104.pdf
SA系統分析師_李小華_104.pdf
SA系統分析師_陳志明_104.pdf
```

## 9. 狀態顯示說明

Popup 中的狀態區會顯示：

- `running`：目前是否執行中
- `progress`：進度，例如 `2/10`
- `current`：目前處理到第幾份 / 總數
- `last`：最後一筆狀態訊息

工具列 Badge 會顯示：

- `PDF`：待命
- `1/5`、`2/5`：批次進行中
- `OK`：完成
- `ERR`：有錯誤

## 10. 常見問題排查

## 10.1 點下載沒反應

請先確認：

1. 目前頁面是否為 104 可辨識的履歷頁
2. 你是否已登入 104 VIP
3. 擴充功能是否已成功安裝
4. 該頁面是否載入完成
5. 重新整理頁面後再試一次

---

## 10.2 批次中途停住或只下載幾份

可能原因：

- 中途遇到第一個非履歷頁，所以流程正常停止
- 某些頁面尚未載入完成
- Chrome 跳出下載詢問，阻斷自動流程
- 網路較慢，等待時間設定太短

建議做法：

- 關閉「每次下載前詢問儲存位置」
- 提高 `首份等待` 與 `後續等待`
- 把要下載的頁面集中在同一個視窗中
- 先確認每個分頁都能正常打開

---

## 10.3 下載的 PDF 沒有出現在指定位置

請檢查：

1. Chrome 預設下載資料夾設定是否正確
2. Popup 中的「下載子資料夾」是否填寫正確
3. 子資料夾路徑是否使用可接受的格式
4. 是否被系統或瀏覽器安全機制擋下下載

注意：這個欄位是 **相對於 Chrome 預設下載資料夾**，不是輸入完整硬碟絕對路徑。

---

## 10.4 檔名不如預期

檔名通常是根據履歷姓名組合而成。若候選人姓名讀取失敗，可能會改用其他可辨識值。  
建議：

- 確認履歷預覽頁可正常開啟
- 避免頁面尚未完全載入就執行
- 拉高等待時間

---

## 10.5 為什麼第一份比較慢？

這是正常的。第一份通常需要：

- 初始化背景分頁
- 等待履歷預覽完整載入
- 啟動 PDF 匯出流程

因此首份等待時間通常會設得比後續長。

## 11. 權限說明

此套件會使用以下權限：

- `tabs`：讀取與切換分頁
- `scripting`：從分頁取得頁面資訊
- `downloads`：下載 PDF 檔案
- `debugger`：呼叫瀏覽器的 PDF 匯出能力
- `storage`：保存設定與進度狀態

主機權限包含：

- `https://vip.104.com.tw/ResumeTools/resumePreview?*`
- `https://vip.104.com.tw/search/*`
- `https://vip.104.com.tw/document/*`

## 12. 更新方式

拿到新版時：

1. 用新版檔案覆蓋原本的擴充功能資料夾
2. 打開 `chrome://extensions/`
3. 找到此套件後點 **重新載入**
4. 回到 104 頁面重新整理即可

若更新後出現異常，建議直接移除舊版再重新安裝。
